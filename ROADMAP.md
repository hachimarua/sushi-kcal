# すしkcal ロードマップ

## 目的

スシロー・くら寿司での外食時に、食べた商品の合計カロリーをすばやく出し、MealTracker の QuickAdd に「名前」と「kcal」だけ手入力できるようにする。

MealTracker 側では塩分は入力しない運用とし、寿司アプリ側でも PFC・塩分は扱わない。

## Vol.0.1 MVP

状態: 完了

- スシロー / くら寿司のカロリーデータを同梱
- ネタ名検索
- タップで皿に追加
- 個数変更
- 合計 kcal 表示
- MealTracker 転記用に「名前」「kcal」「まとめ」をコピー
- PWA 用の manifest と service worker を追加

## Vol.0.2 店舗専用モード

状態: 完了

- 起動時に「スシロー」「くら寿司」を選ぶ
- 選択後はその店専用の電卓として表示する
- 店舗をまたいだ合算はしない
- 初期画面は「いらっしゃいませ!」と暖簾背景で、寿司屋に入る雰囲気を出す
- 店名は「版」を外し、スシロー / くら寿司の看板風カードとして表示する
- 「今日の皿」はレーン背景と丸皿の積み上げで、皿として分かる見た目にする
- 同名商品は代表行に統合する
- 代表行は価格欄があるものを優先する
- 検索候補の表示を外食中に見やすく整理する

## 判断メモ

- はしご利用は想定しないため、店舗別カートを画面上で同時表示しない
- データはオフライン同梱を基本にする
- PFC・塩分は MealTracker 側の QuickAdd 表示に任せる
- 新しい外部サービスは増やさず、まず静的PWAとして運用する
