# Vol.0.3 iPhone URL運用メモ

## 目的

iPhone で実際に開き、ホーム画面追加と MealTracker 転記の流れを確認する。

## 今すぐ試すURL

Mac と iPhone が同じ Wi-Fi にいる場合は、まず以下を試す。

```text
http://Air.local:8765/?v=11
```

表示できない場合は、Mac の Wi-Fi IP アドレスを確認して、以下の形で開く。

```text
http://<MacのIPアドレス>:8765/?v=11
```

例:

```text
http://192.168.1.23:8765/?v=11
```

この方法は同じ Wi-Fi 内の確認用。外出先では使えない。

## ホーム画面追加の注意

- Safari でURLを開く。
- 共有メニューから「ホーム画面に追加」を選ぶ。
- 表示名が「すしkcal」、アイコンがまぐろ・たまご・ガリになっているか確認する。
- 画面更新が反映されない場合は、URL末尾に `?v=11` を付けて再読み込みする。

## 外出先で使うURL

実運用では HTTPS の静的ホスティングに置くのがよい。

候補:

- GitHub Pages
- Cloudflare Pages

現時点では、新しい外部サービスを増やしすぎない方針なので、まずは既存の GitHub 運用があるなら GitHub Pages を優先する。

## Vol.0.3 の判定

同じ Wi-Fi で実機確認し、UIとMealTracker転記に問題がなければ、次に外出先URL化へ進む。

外出先URL化まで済み、スシローまたはくら寿司で1回使えたら Vol.0.3 完了とする。
